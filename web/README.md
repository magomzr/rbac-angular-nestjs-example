# RBAC - Web flow

> NOT generated by AI. These are my study notes.

### 1. Starting app - ONE single time.

1. **main.ts** - Bootstrap everything:
   - Calls `bootstrapApplication()` with `AppComponent` and `appConfig`,
   - Starts the Angular app.

2. **app.config.ts** - App configuration:
   - Registers `provideRouter(routes)`,
   - Registers `provideHttpClient()` with `authInterceptor`.
   - All HTTP requests will NOW include the JWT token automatically

3. **AuthService constructor** - Executed ONCE:
   - Checks if there's a token in `localStorage`,
   - If exists, parses the JWT and extracts the payload,
   - Converts `permissions[]` array to `Set<string>` and stores it in memory,
   - This Set is reused for all permission checks. Faster.

### 2. Flow per Route Navigation.

1. **authGuard** (route protection)

- **O(1)**,
- Checks if user is logged in with `auth.isLoggedIn()`,
- If so, allows navigation.
- If not logged in, redirects to `/login` then.

2. **permissionGuard(permission)** (optional route protection)

- **O(1)**,
- Checks if user has specific permission with `auth.hasPermission()`,
- Uses `Set.has()` which is O(1),
- If no permission, redirects to `/forbidden`,
- Otherwise, allows navigation

### 3. Flow per Component Render.

1. **CanDirective** (`*appCan`)

- **O(1)**,
- Used in templates like `<button *appCan="'delete:element'">Delete</button>`,
- Executed during component initialization,
- Calls `auth.hasPermission()` which uses `Set.has()`,
- If user has permission, creates the embedded view (renders the element),
- If no permission, does nothing (element not rendered in DOM).

2. **AuthService.hasPermission()**

- **O(1)**,
- Simply returns `this._permissions().has(permission)`,
- Set lookup is constant time,
- No loops, no iterations.

### 4. Flow per HTTP Request.

1. **authInterceptor**

- **O(1)**,
- Intercepts every HTTP request,
- Gets token from `AuthService.getToken()`,
- If token exists, clones the request and adds `Authorization: Bearer ${token}` header,
- Passes the request to the next handler.
- Render and finish
